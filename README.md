## Política de Versionamento

- **Padrão:** Semantic Versioning (X.Y.Z)
- **Branches:**
  - `main`: versão estável
  - `dev`: desenvolvimento
  - `feature/*`: novas funcionalidades
  
- **Commits:** Mensagens em português, formato: 
  `tipo(escopo): descrição`
  Exemplo: `docs(requisitos): adiciona RF01 e RF02`

- **Tags:** Versões liberadas seguem padrão v1.0.0, v1.1.0, etc.
